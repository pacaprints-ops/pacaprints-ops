# SEO Audit Report — PacaPrints (www.pacaprints.com)
**Date:** 2026-03-12

---

## Summary

The site has a solid technical foundation — sitemap, robots.txt, canonical tags, and product JSON-LD are all working correctly. However there are two critical gaps costing rankings: the homepage has **no meta description** and **no H1**, and **Open Graph tags aren't rendering** on the homepage (no social share preview). Most critical fixes are admin-level changes, no code needed.

## Issues

### ❌ Critical

**1. No meta description on the homepage**
The live site returns no `<meta name="description">`. The theme only outputs it when `page_description` is set (`snippets/meta-tags.liquid` ~line 116). It's not set in Shopify Admin.
**Fix:** Admin → Online Store → Preferences → Homepage meta description (aim for 140–155 chars).

**2. No H1 on the homepage**
Zero `<h1>` tags in the live homepage HTML. The hero section renders headings via theme editor blocks — the H1 block appears to have been removed or changed.
**Fix:** Theme editor → Homepage → confirm the hero/first section has a text block set to H1.

**3. Open Graph tags not rendering on homepage**
No `og:title`, `og:description`, or `og:image` on the live homepage — every social share shows a blank preview card.
- `og:description` needs the shop description set in Admin → Settings → General
- `og:image` only outputs when a Social sharing image is configured (`snippets/meta-tags.liquid` ~line 55)

**Fix:** Admin → Online Store → Preferences → upload a Social sharing image (1200×630px). Also verify Admin → Settings → General → Store description is filled in.

---

### ⚠️ Warnings

**4. Homepage title too long**
Live title is 83 characters — over the ~60 char display limit. Shorten in Admin → Online Store → Preferences.

**5. `og:image` uses `http:` not `https:`**
`snippets/meta-tags.liquid` ~line 58: `content="http:{{ page_image | image_url }}"` — the protocol should be `https:`.

**6. Missing `alt` on image_tag calls**
- `snippets/link-featured-image.liquid` lines 19, 27, 37, 45 — nav images, no `alt:`
- `snippets/resource-card.liquid` lines 70–88, 107 — product card images, no `alt:`
- `snippets/image.liquid` line 29 — generic image snippet, no `alt:` passed through

**7. `target="_blank"` missing `noreferrer`**
| File | Line |
|------|------|
| `blocks/icon.liquid` | 21 |
| `sections/hero.liquid` | 239 |
| `templates/gift_card.liquid` | 207 |
Change `rel="noopener"` → `rel="noopener noreferrer"` in each.

**8. Minimal Organization JSON-LD**
`sections/header.liquid` lines 159–169 outputs only name, logo, url. Missing `sameAs` (social profiles) and `contactPoint`.

**9. No BreadcrumbList JSON-LD**
No breadcrumb structured data on product or collection pages — missed opportunity for enhanced SERP display.

## Recommended Fixes (priority order)

**No code needed — Admin changes first:**
1. Set homepage meta description (Admin → Online Store → Preferences)
2. Upload a Social sharing image (same location) — fixes missing `og:image` everywhere
3. Fill in Store description (Admin → Settings → General) — fixes `og:description`
4. Add H1 to homepage in theme editor
5. Shorten homepage title to under 60 chars

**Code fixes:**
6. `snippets/meta-tags.liquid` ~line 58 — change `http:` to `https:` for `og:image`
7. Add `alt:` to image_tag calls in `link-featured-image.liquid`, `resource-card.liquid`, `image.liquid`
8. Add `noreferrer` to the 3 `target="_blank"` links above
9. Enhance Organization JSON-LD with `sameAs` social links
10. Add BreadcrumbList JSON-LD to product/collection sections
